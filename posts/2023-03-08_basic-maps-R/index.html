Required packages

  • {sf} (simple features): is a package that . Installing it may be tricky1, as it relies on other libraries. It supercedes sp (see below)

  • {sp}: provides classes and methods for spatial data, just as sf does. Despite sf superseedes sp in many ways, as of today, there are still lots of documentation mentioning them, and many other packages (such as tmap or spData -see below) will install them as a dependency.

  • Data:

    • some packages provide default datasets (i.e. tmap comes with world),

    • using packages that provide spatial datasets_

      • {spData} provides several datasets (most of them are from the USA and the World) in different R spatial objects either vectorial (sf and its predecessor sp) or raster.

      • https://github.com/Nowosad/spDataLarge

      • {usmap} maps for the USA

    • read it from geospatial files (geopackage, json or shapefile2) or using packages that pro

      • https://www.census.gov/geographies/mapping-files/time-series/geo/cartographic-boundary.2021.html

  • Visualisation

    • base r (plot)

    • ggplot

    • tmap: https://github.com/r-tmap/tmap

    • https://cran.r-project.org/web/packages/maps/maps.pdf

  • 2 Shapefile is a very popular file format, but it is also very old, very different (it actually consists of several files with different extensions, which is confusing for many newbies) and, above all, controversial. Unsurprisingly, there are a number of websites explaining the reasons for not using it, such as the popular: http://switchfromshapefile.org/
